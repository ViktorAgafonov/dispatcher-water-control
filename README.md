# Диспетчер Водоснабжения

Программно-аппаратный комплекс для оперативного контроля и диспетчеризации водоснабжения производственных объектов. Обеспечивает сбор и обработку данных со счётчиков Пульсар и Сигнур, отображение текущих объёмов и расходов воды, учёт сточных вод, а также формирование статистики и баланса водопотребления в режиме реального времени.

> **Версия:** 1.1.0 · **Платформа:** Electron + Node.js · **ОС:** Windows

---

## Возможности

- **Водоснабжение** — опрос счётчиков Пульсар (протокол RS-485 через TCP-конвертер), мнемосхема с актуальными показаниями, расход и объём по каждому узлу.
- **Водоотведение** — опрос счётчиков Эхо Сигнур (Modbus RTU через конвертер MOXA RS-232/TCP), отдельная мнемосхема сточных вод.
- **Статистика** — почасовая история (~8 суток), суточный и недельный расход с выбором начала периода.
- **Отчёты** — сводные карточки (вход/выход/потери) и таблица истории потребления (24 ч / 7 дней), экспорт в CSV (UTF-8, совместим с Excel).
- **Логи** — кольцевой буфер, фильтры по уровню и источнику, автопрокрутка.
- **Трей** — приложение работает в фоне, иконка в системном лотке с подсказкой по каждому счётчику.

## Сборка дистрибутива (Windows x64)

```bash
npm run build        # dist\Диспетчер Водоснабжения-win32-x64\
npm run build:zip    # + dist\ДиспетчерВодоснабжения-1.1.0-win-x64.zip
```

Или через готовые обёртки:
```bat
build.bat
build.bat zip
```
```powershell
.\build.ps1
.\build.ps1 -Zip
```

Результат — портативный каталог (~95 МБ), запускается без установки.

## Настройки

Хранятся в `config.json` в каталоге данных приложения (`%APPDATA%\dispatcher-water-control\`). При первом запуске создаются значения по умолчанию.

### Счётчики водоснабжения (Пульсар)
| Поле | Описание |
|---|---|
| Название | Произвольное (например, «Цех 1») |
| IP / Порт | Адрес TCP-конвертера RS-485↔Ethernet |
| Адрес | Серийный номер прибора (BCD) |
| Сут. с ч. | Час начала суток (0–23) |
| Нед. с дн. | День начала недели (1=Пн … 7=Вс) |

### Счётчики водоотведения (Эхо Сигнур)
| Поле | Описание |
|---|---|
| Название | Произвольное (например, «Колодец 1») |
| IP / Порт | Адрес конвертера MOXA (режим TCP-Server) |
| Адрес | Modbus-адрес прибора (1–247) |

## Структура проекта

```
main.js              — главный процесс: опрос, IPC, трей, статистика
preload.js           — безопасный мост renderer ↔ main
src/
  pulsar.js          — протокол Пульсар-М (RS-485, CRC16, BCD, double64 LE)
  sigur.js           — протокол Эхо Сигнур (Modbus RTU fn=0x03 over TCP)
  tcpClient.js       — TCP-клиент с таймаутом и буферизацией
  stats.js           — почасовая история, суточный/недельный расход
  logger.js          — кольцевой лог с событиями
renderer/
  index.html         — структура UI (вкладки)
  renderer.js        — логика UI, мнемосхемы, отчёты
  styles.css         — стили
scripts/
  post-build.js      — очистка локалей после packager
  build-zip.js       — упаковка в ZIP
```

## Протоколы

| Счётчик | Интерфейс | Протокол | Конвертер |
|---|---|---|---|
| Пульсар-М | RS-485 | Собственный (fn=0x01, CRC16, BCD) | RS-485↔Ethernet (производитель) |
| Эхо Сигнур (ЭХО-Р-03) | RS-232 | Modbus RTU (fn=0x03, 10 рег.) | MOXA, режим TCP-Server |
