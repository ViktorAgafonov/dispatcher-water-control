Используется только режим RTU и специальные команды (User-Defined Function Codes, диапазон допустимых командных
кодов в десятичной системе счисления – от 100 до 110).
Каждый командный фрейм должен включать адрес подчинённого (1 байт), код команды (1 байт), параметры (от 0 до 3 байтов) и
CRC-код (2 байта; первым передаётся младший байт). Команды, после которых в скобках стоит обозначение К2, относятся ко второму
каналу прибора ЭХО-Р-03-2.

Доступные для чтения по команде 03 адреса регистров и их содержание:
Команда контроллера код - 102 (Передать текущие результаты измерений)

Ответ прибора:
Адрес - 1Б
Команда - 1Б
Число байтов - 1Б
Тип прибора - 1Б (ЭХО-Р-03-1 - 07, ЭХО-Р-03-2 - 08, ЭХО-Р-03-3 - 09)
Версия ПО - 1Б
Заводской номер - 2Б, word
Служебные байты - 2Б
CRC - 2Б

ПРИМЕР
запрос текущих значений у прибора ЭХО-Р-03-1 с адресом 1: 01 66 80 0А
ответ: 01 66 12 9A 99 99 3E 54 DF 4B 3D 6D 00 04 00 7A 7C 00 00 02 00 81 18
значение расхода занимает с 8 по 11 байты: 54 DF 4B 3D
поскольку многобайтные коды передаются младшим байтом вперед, то переводим ответ в нормальную последовательность: 3D 4B DF 54.
это число с плавающей точкой (IEEE754), в десятичном виде 0,04978125 [куб.м/сек] или 0,04978125 * 3600 = 179,2 куб.м/ч
значение объема занимает с 12 по 15 байты: 6D 00 04 00, множитель находится в 20 байте: 02
поскольку многобайтные коды передаются младшим байтом вперед, то переводим ответ в нормальную последовательность: 00 04 00 6D.
в десятичном виде 262253
множитель: 10 ^ (02 - 3) = 0,1
объем: 262253 * 0,1 = 26225,3 [куб.м]
время учета расположено в байтах с 16 по 19: 7A 7C 00 00
поскольку многобайтные коды передаются младшим байтом вперед, то переводим ответ в нормальную последовательность: 00 00 7С 7А.
в десятичном виде: 31866 [мин], что соответсвует 531 часу 6 минутам.
код неисправности находится в 21 байте: 00, что говорит об исправном приборе. Расшифровка кодов неисправности приведена в руководстве по эксплуатации.

Адреса регистров, доступных для чтения по команде 03:
0х0000 – 0х0001 -> H(float) уровень, м
0х0002 – 0х0003 -> Q(float) расход, м3/сек
0х0009		-> Pu(byte) lg(KU) – 3, KU – масштабирующий множитель объёма

ПРИМЕР
Запрос значения счетчика объема у прибора с адресом 01: 01 03 00 04 00 06 84 09
Ответ: 01 03 0С E5 C3 04 00 17 82 00 00 18 A4 03 0E 8B 85
значение счетчика объема в данном ответе занимает с 4 по 7 байты: E5 C3 04 00, множитель в 14 байте: 03
поскольку многобайтные коды передаются младшим байтом вперед, то переводим ответ в нормальную последовательность: 00 04 С3 Е5.
в десятичном виде 312293
множитель: 10 ^ (03 - 3) = 1
объем: 312293 * 1 = 312293 [куб.м]


Ниже приведено формирование запросов для обоих случаев на Node.js:

1. Формирование запроса (Команда 102 / 0x66)
// Запрос текущих результатов измерений (Код 102 / 0x66)
const unitId = 0x01;
const commandCode = 0x66; // 102 в десятичной
const parameter = 0x00;    // Параметр (согласно примеру 01 66 80 0A, где 80 - параметр)

const request = Buffer.alloc(4); 
request.writeUInt8(unitId, 0);
request.writeUInt8(commandCode, 1);
request.writeUInt8(0x80, 2); // Параметр из примера

// Расчет и добавление CRC (младшим байтом вперед)
const crc = calculateCRC16(request.slice(0, 3));
request.writeUInt16LE(crc, 3); 

// Итог для порта: <Buffer 01 66 80 0a>

2. Формирование запроса (Команда 03)
// Запрос значения счетчика объема (Код 03)
const unitId = 0x01;
const functionCode = 0x03;
const startRegister = 0x0004; // Адрес из примера для объема
const count = 0x0006;        // Количество регистров из примера

const request = Buffer.alloc(8);
request.writeUInt8(unitId, 0);
request.writeUInt8(functionCode, 1);
request.writeUInt16BE(startRegister, 2);
request.writeUInt16BE(count, 4);

const crc = calculateCRC16(request.slice(0, 6));
request.writeUInt16LE(crc, 6);

// Итог для порта: <Buffer 01 03 00 04 00 06 84 09>

3. Обработка ответа (Команда 102)
Разбор специфического бинарного пакета «Сигнур» с учетом порядка байтов (Little-Endian) и множителя.
function parseResponse102(buffer) {
    // 1. Текущий расход (байты 8-11, IEEE754, Little-Endian)
    // В примере: 54 DF 4B 3D -> 0.04978...
    const flowM3Sec = buffer.readFloatLE(7); 
    const flowM3Hour = flowM3Sec * 3600;

    // 2. Суммарный объем (байты 12-15, UInt32, Little-Endian)
    const rawVolume = buffer.readUInt32LE(11);
    
    // 3. Множитель объема (байт 20)
    const multiplierRaw = buffer.readUInt8(19);
    const multiplier = Math.pow(10, (multiplierRaw - 3));
    
    const totalVolume = rawVolume * multiplier;

    return {
        flow: flowM3Hour.toFixed(2),
        volume: totalVolume.toFixed(1),
        unit: "m3"
    };
}

CRC16: В обоих случаях используется алгоритм CRC16-Modbus, где результат записывается в буфер как writeUInt16LE (младший байт первым).Смещение индексов: В buffer.read... индексы начинаются с 0, поэтому 8-й байт в документации соответствует индексу 7 в коде.Команда 102: Это самый эффективный способ, так как он сразу отдает код неисправности (21-й байт), что критично для достоверности данных.
